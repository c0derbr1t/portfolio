What is this?

Essentialism involves doing less, but better, so you can make the highest possible contribution. It's about getting the right things done and regaining control of our own choices and time-management. This application allows you to identify values important to you, and then choose the top 3 of those. You complete a prompt on why those 3 values are important. Then you can add projects/things you are currently involved in, and compare those items with your values.

What kind of build was this?

This was a remote, cross-functional team of 5 over 1 week. I worked as the sole Back-end Developer, as well as the Project Manager for my team.

What did I do?

  • As Back-End Developer:
  • I used DB Designer to design the database schema.
  • I set up the backend. I used xpress, helmet, and cors in the server. I made routers for authorization, users, values, prompts, and projects. I also created restricted middleware to only allow access to most of the routes while logged in.
  • In the authorization router, I used bcrypt and tokens to encrypt passwords during registration, and to check passwords during login.
  • I set up a .env file for secrets. Then made the dbConfig file using knex. I used knex migrations to create the tables for my database, and then I created seeds for each table.
  • I created models and routes for each section.
  • I created tests using jest and supertest
  • I used sqlite3 for development and testing, and postgres for production.
  • I created a very detailed Readme for the API including all routes, expected input and output, and error codes.
  • As Project Manager:
  • I created the repositories for the back-end, front-end, and marketing page.
  • I created the Trello Board.
  • I controlled the slack channel, and made sure my team had the information they needed accessible.
  • I helped our team be cohesive, and led discussions to help with team connectivity and morale.
  • I led daily stand-ups with my team to ensure the whole team was on the same page, and we were all aware of what had been done, and what still needed accomplished.
  • I celebrated accomplishments with team-members.
  • I provided constructive feedback on behavior and code.
  • I communicated with each team-member's Team Lead if neccessary.
  • I completed a retrospective at the end of the week, and review of each team member.
  • I assisted each person with their programming as neccessary, in a way that would help them understand and grow.
